# Задание: Контекстные менеджеры и управление ресурсами

## 🎯 Цель и краткое описание

Реализовать **пул ресурсов** с безопасным выдачей/возвратом через контекстный протокол и при необходимости **`contextlib.ExitStack`**.

## 📚 Темы для изучения/повторения

- Контекстные менеджеры: `__enter__` / `__exit__` или `@contextmanager`
- `contextlib.ExitStack` — несколько ресурсов с гарантированным закрытием
- RAII-подобный паттерн в Python: получил ресурс — вернул в пул в `finally`
- Исключения в `__exit__`: не подавлять лишнее без причины

## 📝 ТЗ

**Реализовать в `solution.py`:**

1. Класс `Resource` с полем `id: int` и методом `close()` (устанавливает флаг `_closed`).
2. Класс `ResourcePool` с фиксированным набором `Resource` (передаётся в конструктор списком).
3. Метод `acquire() -> contextmanager` — контекстный менеджер: при входе выдаёт свободный ресурс (если нет — ждать можно **синхронно** через `queue.Queue`, без asyncio); при выходе возвращает ресурс в пул; при исключении внутри блока ресурс всё равно возвращается.
4. Функция `with_resources(pool, k: int)` — используя `ExitStack`, открыть `k` вложенных `acquire()` и вернуть список из `k` разных ресурсов; при выходе все возвращены.

**Требования:**

- `contextlib`, `queue`, `typing`.
- Один поток (без `threading` для конкуренции — только структура пула).

## ✅ Как проверить

```bash
cd tasks/task_06_context_resource_pool
python -c "
from solution import Resource, ResourcePool, with_resources

r1, r2 = Resource(1), Resource(2)
pool = ResourcePool([r1, r2])

with pool.acquire() as a:
    with pool.acquire() as b:
        assert a.id != b.id

out = with_resources(pool, 2)
assert len({x.id for x in out}) == 2
print('OK')
"
```

## 💡 Подсказки и ограничения

- **Не открывать** файлы ОС — абстрактный `Resource` достаточен.
- **Акцент:** повторное использование одного и того же объекта после возврата в пул.
- **Типичная ошибка:** забыть вернуть ресурс при исключении — используйте `@contextmanager` с `try/finally`.
