# Задание: Метакласс, профилирование, упаковка

## 🎯 Цель и краткое описание

Зарегистрировать подклассы через **метакласс**, добавить простой **счётчик вызовов** (как у лёгкого профилировщика) и минимальный **`pyproject.toml`** для пакета без зависимостей.

## 📚 Темы для изучения/повторения

- Метаклассы: `__init_subclass__` **или** `type` как метакласс для реестра классов
- `functools.wraps` при обёртке `__init__` для подсчёта созданий
- `cProfile` / `pstats` — снять профиль на коротком скрипте (в README — команда)
- Структура пакета: `pyproject.toml` с `[build-system]` и `[project]` (PEP 621)

## 📝 ТЗ

**Код в пакете `registry_pkg/`:**

1. Метакласс `RegisteredMeta`, который ведёт класс-атрибут `registry: dict[str, type]` — имя класса → класс для всех наследников `Base` (кроме самого `Base`).
2. Класс `Base(metaclass=RegisteredMeta)` и два подкласса `A`, `B`.
3. Декоратор `@trace_init` для функций/методов: увеличивает глобальный словарь `CALL_COUNTS: dict[str, int]` по ключу `qualname` при каждом вызове обёрнутого callable (повесьте на `__init__` класса `A`).

**Корень задания:**

4. Файл `pyproject.toml` с именем проекта `registry-pkg`, версия `0.1.0`, пакет находится в `registry_pkg`.

5. В `README` этой папки (данном файле) уже указана команда профилирования — создайте `demo_profile.py`, который создаёт 1000 экземпляров `A` и запускается под `python -m cProfile`.

## ✅ Как проверить

```bash
cd tasks/task_10_metaclass_registry
python -c "
from registry_pkg.base import Base, A, B, CALL_COUNTS

assert 'A' in Base.registry and 'B' in Base.registry
x = A()
assert CALL_COUNTS.get('A.__init__', 0) >= 1
print('OK')
"
```

Профилирование (пример):

```bash
python -m cProfile -s cumtime demo_profile.py | head -20
```

(На Windows вместо `head` используйте перенаправление в файл или `Select-Object -First 20`.)

## 💡 Подсказки и ограничения

- **Не злоупотреблять** метаклассами в проде — здесь учебная демонстрация реестра.
- **Акцент:** `__init_subclass__` часто проще метакласса — для задания требуется именно **метакласс** по ТЗ.
- **Типичная ошибка:** задвоение имён в `registry` при перезагрузке модулей в REPL — в тестах не критично.
