# Задание: Платёжная система (ООП)

## 🎯 Цель и краткое описание

Спроектировать расширяемую обработку платежей: разные способы оплаты подключаются через **композицию** (стратегии), общий контракт задаётся через **ABC**, соблюдение **SOLID** (особенно OCP и DIP).

## 📚 Темы для изучения/повторения

- `abc.ABC` и `@abstractmethod` — контракт без жёсткой иерархии реализаций
- Композиция: класс «оркестратор» держит стратегию, а не наследует «все способы сразу»
- Разделение: валидация суммы, исполнение платежа, логирование — разные ответственности
- Принцип подстановки Лисков в контексте подмены стратегий одного интерфейса

## 📝 ТЗ

**Реализовать:**

1. Абстрактный базовый класс `PaymentStrategy` с методом `pay(amount: Decimal) -> str` (возвращает идентификатор или сообщение об успехе).
2. Две конкретные стратегии: `CardPayment` и `WalletPayment` (достаточно имитации: строка с префиксом `CARD:` / `WALLET:` и суммой).
3. Класс `PaymentService`, принимающий в конструкторе стратегию и опциональный `logger` (callable `str -> None`, по умолчанию `print`).
4. Метод `process(amount: Decimal) -> str`: при `amount <= 0` выбрасывать `ValueError`; иначе вызывать логгер, затем стратегию.

**Вход/выход:**

- Вход: `Decimal` сумма, стратегия подставляется при создании сервиса.
- Выход: строка от стратегии.

**Требования к коду:**

- Только стандартная библиотека (`decimal`, `abc`, `typing`).
- Без внешних пакетов.
- Файл решения: `solution.py` в этой папке (или пакет `payment/` — на ваш выбор, но импорты должны быть простыми).

## ✅ Как проверить

```bash
cd tasks/task_01_oop_payment_system
python -c "
from decimal import Decimal
from solution import PaymentService, CardPayment, WalletPayment

svc = PaymentService(CardPayment())
assert 'CARD' in svc.process(Decimal('10.00'))

svc2 = PaymentService(WalletPayment(), logger=lambda s: None)
assert svc2.process(Decimal('1')).startswith('WALLET')

try:
    PaymentService(CardPayment()).process(Decimal('-1'))
except ValueError:
    pass
else:
    raise AssertionError('expected ValueError')
print('OK')
"
```

Пример вывода при `print` как логгере: в консоли появятся строки лога перед результатом стратегии.

## 💡 Подсказки и ограничения

- **Не делать** один «God-класс» с `if card elif wallet` — вынесите варианты в стратегии.
- **Акцент:** композиция (сервис **имеет** стратегию), а не глубокое наследование «сервис от карты».
- **Типичная ошибка:** забить на `Decimal` и использовать `float` для денег — для задания используйте `Decimal`.
