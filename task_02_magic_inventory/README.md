# Задание: Инвентарь с протоколами и дескриптором

## 🎯 Цель и краткое описание

Потренировать **магические методы** (`__iter__`, контекстный менеджер) и простой **дескриптор** для валидации атрибута класса.

## 📚 Темы для изучения/повторения

- Протокол итерации: `__iter__` / (опционально) `__next__` для своей коллекции
- Контекстный менеджер: `__enter__` и `__exit__` для временной «блокировки» изменений
- Дескрикторы: `__get__` / `__set__` и хранение значения через `WeakKeyDictionary` или имя в `__dict__`
- Инварианты данных: отклонять некорректные присваивания на уровне дескриптора

## 📝 ТЗ

**Реализовать:**

1. Класс `PositiveInt` — дескриптор: разрешает только целые `> 0` (иначе `ValueError`).
2. Класс `Inventory` с атрибутом `capacity: PositiveInt` и внутренним списком слотов (строки названий предметов или пусто).
3. `Inventory` поддерживает итерацию по непустым слотам в порядке добавления.
4. Метод контекста `locked()` возвращает менеджер: внутри блока `add`/`remove` бросают `RuntimeError`; после выхода — снова разрешено.

**Вход/выход:**

- `add(name: str)` — добавить в первый свободный слот; если переполнение — `ValueError`.
- `remove(name: str)` — удалить первое вхождение; если нет — `ValueError`.

**Требования к коду:**

- Стандартная библиотека + `typing`.
- Файл: `solution.py`.

## ✅ Как проверить

```bash
cd tasks/task_02_magic_inventory
python -c "
from solution import Inventory, PositiveInt

inv = Inventory(capacity=2)
inv.add('sword')
inv.add('shield')
assert list(inv) == ['sword', 'shield']

try:
    inv.add('x')
except ValueError:
    pass
else:
    raise AssertionError('overflow')

with inv.locked():
    try:
        inv.remove('sword')
    except RuntimeError:
        pass
    else:
        raise AssertionError('locked')

inv.remove('sword')
assert list(inv) == ['shield']
print('OK')
"
```

## 💡 Подсказки и ограничения

- **Не путать** итерацию инвентаря с итерацией по «всем слотам включая пустые», если ТЗ просит только непустые.
- **Акцент:** дескриптор переиспользуемый (можно повесить на другой класс).
- **Типичная ошибка:** забыть `exc_type` в `__exit__` — для этого задания подавлять исключения не нужно, просто снимайте флаг блокировки.
