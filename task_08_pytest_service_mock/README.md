# Задание: Тестирование с pytest

## 🎯 Цель и краткое описание

Покрыть сервис **pytest**: **fixtures**, **`unittest.mock`**, **`parametrize`**, изоляция внешних зависимостей.

## 📚 Темы для изучения/повторения

- Фикстуры: scope, переиспользование
- `unittest.mock.Mock` / `MagicMock`, патчинг `patch`
- `@pytest.mark.parametrize` — табличные тесты
- Arrange-Act-Assert, граничные случаи

## 📝 ТЗ

В папке задания:

1. `service.py` — класс `QuoteService` с методом `fetch_word() -> str`: вызывает `provider()` — callable без аргументов, возвращающий строку; возвращает **первое слово** (до пробела) или пустую строку, если результат пустой.
2. `test_service.py` — тесты:
   - Параметризация: входы `"hello world"`, `"single"`, `""` — ожидаемые `"hello"`, `"single"`, `""`.
   - Фикстура `service` с подставленным mock-провайдером.
   - Тест с `patch` на метод, доказывающий, что при исключении провайдера исключение пробрасывается (или оборачивается — выберите одно поведение и задокументируйте в тесте).

**Зависимости:** только `pytest` (установка: `pip install pytest`).

## ✅ Как проверить

```bash
cd tasks/task_08_pytest_service_mock
pip install pytest
pytest -q
```

Пример вывода: `... [100%]` и все тесты зелёные.

## 💡 Подсказки и ограничения

- **Не ходить** в сеть из тестов — только моки.
- **Акцент:** провайдер передаётся в конструктор `QuoteService(provider=...)`.
- **Типичная ошибка:** мок вызывается не тем способом — используйте `Mock(spec=...)`.
